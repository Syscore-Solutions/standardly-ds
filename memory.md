# Project Memory & Context

## User Preferences & Guidelines
- **Single Source of Truth:** The design system tokens in `tokens/` are generated from Figma via Tokens Studio.
- **Component Styling:** React components must reference semantic CSS custom properties (e.g., `var(--color-general-background)`) generated by Style Dictionary. Do not hardcode raw hex codes, spacing pixels, or raw values.
- **Documentation:** Keep documentation (such as `design.md` and `architecture.md`) up to date as components or token pipelines evolve.
- **Workflow Flow:** Plan changes in `plan.md` and seek user review before executing. Track actions in `tasks.md`.

## Current Goals & Tasks
- **POC Validation:** Verify the Figma → Tokens Studio → Style Dictionary → React components → Storybook workflow loop.
- **Components expansion:** Expand core design system components (Button, Accordion, etc.) based on token specifications.

## Key Decisions & Lessons Learned
- **Two-Layer Token Model:** Decided on a strict separation of Primitive tokens (raw palettes) and Semantic tokens (theme-aware components) to easily support Light/Dark mode transitions.
- **Agent Coordination:** Established `plan.md` and `tasks.md` as repo-level coordination workspaces to prevent different agents from stepping on each other's toes.
- **GitHub identity is repo-level:** GitHub username (`naman-sharma-syscore`) in package.json URLs is repo configuration, not personal branding — retained as-is.
- **Case sensitivity matters:** macOS hides `DESIGN.md` vs `design.md` mismatches; always use `git rm --cached` to fix tracked-name casing.
- **Markdown lowercase standardization:** Standardized `README.md` to lowercase `readme.md` in both Git tracking and documentation references.
